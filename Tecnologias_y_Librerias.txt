REPORTE DE TECNOLOGÍAS Y LIBRERÍAS
PROYECTO: YOUTUBE DOWNLOADER PRO
--------------------------------------------------

1. ¿USAMOS LIBRERÍAS?
--------------------------------------------------
SÍ. Aunque no se vea a simple vista, el programa se apoya en "Cajas de Herramientas" (Librerías) muy potentes para funcionar. No reinventamos la rueda, la ensamblamos.

2. LIBRERÍAS EXTERNAS (DEPENDENCIAS)
--------------------------------------------------
Estas son las piezas de Lego que descargamos de internet (NPM) para construir la app:

A) PRINCIPALES (Producción)
   - electron: El "navegador" que permite que tu código web (HTML/JS) se comporte como un programa de escritorio (.exe).
   - express: Es un servidor web miniatura. Lo usamos dentro de la app para gestionar las peticiones (descargar, buscar info, pausar).
   - cors: Un "segurata" que permite que el Frontend (la parte visual) hable con el Servidor interno sin bloqueos de seguridad.

B) HERRAMIENTAS DE CONSTRUCCIÓN (Desarrollo)
   - electron-builder: El "empaquetador". Coge todo el código suelto y lo comprime en el archivo instalador (.exe) final.

3. MOTORES "INVISIBLE" (BINARIOS)
--------------------------------------------------
Estos no son código JavaScript, son programas completos que tu app controla "por detrás":

   - yt-dlp.exe: El MEJOR descargador del mundo (Open Source). Tu app le da órdenes ("baja este link en MP3") y él hace el trabajo duro.
   - ffmpeg.exe: El editor de video/audio. Se usa para convertir lo que baja yt-dlp (videos webm/mkv) a formatos estándar (MP3/MP4) que Windows entienda.

4. CÓDIGO FUENTE (LENGUAJES)
--------------------------------------------------
Lo que hemos escrito nosotros a mano:

   - HTML5: La estructura (El esqueleto de la ventana).
   - CSS3 (Vanilla): El diseño. No usamos Bootstrap ni Tailwind. Todo el estilo "Azul/Negro" lo escribimos desde cero (style.css).
   - JavaScript (Node.js): La lógica del servidor (server.js). Maneja archivos, carpetas y ejecuta los motores.
   - JavaScript (DOM): La lógica visual (script.js). Lo que pasa cuando haces click en un botón.

RESUMEN:
Es una aplicación "Full Stack" metida en una caja:
Frontend (Web) + Backend (Node Server) + Motores (yt-dlp) = App de Escritorio.
